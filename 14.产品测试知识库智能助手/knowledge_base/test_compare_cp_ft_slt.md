# CP/FT/SLT 对比知识库：从晶圆到系统的测试链条

半导体测试是层层递进的质量防线，环环相扣，不同阶段的测试对象、设备、成本与目的各异。对PTE（Product Test Engineer）新人而言，最先要理清的便是CP（Chip Probing）、FT（Final Test）和SLT（System Level Test）三者的定位。这三者构成“CP→FT→SLT”的完整测试链：CP筛掉80%的制造缺陷坏片，FT确保封装后的芯片满足规格，SLT则捕捉真正跑系统才会暴露的交互性故障。

## 1. 对比总表

| 测试环节 | 测试对象 | 核心目的 | 典型设备 | 测试时间 | 相对成本占比 | 是否可省略 |
|----------|----------|----------|----------|----------|-------------|-------------|
| **CP** | Wafer上的裸Die | 筛除前道制造缺陷，避免后续封装浪费 | Prober + 探针卡 + ATE | 秒级/Die | 低 | 部分数字逻辑芯片可省略（封装成本极低时），但通常不可省 |
| **FT** | 封装完成芯片 | 验证封装后功能与性能，确保规格达标 | Handler + Socket + Load Board + ATE | 秒～分钟级 | 约占总测试成本35% | 一般不可省略，为出货质量门 |
| **SLT** | 封装芯片在实际系统板上 | 捕捉系统级交互故障（如跨电压域、时钟域、DDR应力、协议一致性） | 系统板 + 运行真实负载 + 温控 | 分钟～小时级 | 极高，专项预算 | 可选，AI/HPC/车规等常必选；消费电子可择机 |

## 2. 各环节典型测试内容展开

### 2.1 CP——晶圆级筛选
CP在晶圆切割前完成，探针卡接触Die的Pad。测试内容偏基础但高效：
- **连续性/漏电（Open/Short）**：检测引脚与探针接触，剔除ESD损伤等。
- **DC参数**：阈值电压、输入漏电、输出驱动能力等，初步验证器件是否“活的”。
- **功能向量**：短小的功能Pattern，快速判断逻辑核心是否能工作。
- **Trim/修调**：对基准电压、内部振荡器等进行烧写或熔丝调整。
- **冗余修复**：存储器内建自修复（BISR）利用冗余行/列替换坏单元。
- 输出：Wafer Map，标记各Die的Good/Bad，实现KGD（Known Good Die）交付。

### 2.2 FT——封装后的最终把关
芯片封装后的电性环境、热学和机械应力都有变化，FT是产品出货前的标准保障：
- **完整Open/Short**：检查所有外部引脚，含引线键合与基板连接。
- **全速功能测试**：应用层功能向量覆盖，验证逻辑时序。
- **性能窗口测试**：频率、延迟、功耗等分别在电压/温度窗口下测试。
- **模拟/混合信号**：ADC/DAC线性度、电源纹波抑制等。
- **Burn-in/应力选项**：部分产品在FT加入短时老化，剔除早期失效。
- 输出：STDF（Standard Test Data Format）数据，用于良率分析、动态分Bin和可追溯。

### 2.3 SLT——系统级回归
SLT让芯片在本就不该妥协的真实系统环境下运行：
- **系统上电与固件加载**：验证Boot、DDR训练、PCIe链路建立等序列。
- **复杂任务场景**：跑持久化的计算任务、神经网络推理、存储压力。
- **跨IP交互验证**：GPU与内存控制器、高速SerDes长时间无错误传输。
- **协议一致性**：USB/PCIe/DDR协议层的电气和逻辑眼图余量。
- **电压/时钟域交叉**：模拟真实平台下多路电源上电顺序及动态跳频。

## 3. PTE新人须知：如何权衡与建议

- **何时必须做CP？** 当晶圆工艺成熟度低、封装成本高（如Flip-Chip BGA）或需要交付KGD用于SiP/MCP时，CP必不可少。如果Die面积小、封装价格极低且流片良率高，可考虑CP节约，但需接受可能良率下降风险。
- **FT的成本与覆盖平衡**：FT成本占比约35%，增加测试时间直接抬高单位成本。PTE需在“抓够缺陷”与“不浪费测试秒数”间博弈，常用产出分析（各测试Item的DPPM贡献）决定是否裁剪。多Site并行测试是降低均摊测试成本的主要手段。
- **SLT投入的合理性**：SLT成本极高，通常按产品预算与市场定位专案申请。AI训练芯片、自动驾驶处理器、服务器CPU等关键任务芯片，靠FT无法覆盖实际工作负载下的跨模块失效，会设定专门的SLT screening。SLT不是“全测”，而是基于抽检或初期全测扫清系统缺陷后放宽。
- **总体逻辑**：CP做粗筛，让封装不浪费在已死的Die上；FT建立在硬件层面“是好的”这一承诺；SLT只做系统能力补给。三者配合，才是完整的质量防线。

## 4. 行业观察：SLT能额外筛掉多少DPPM？

没有SLT的产品，在客户端可能发现若干“测试逃逸”。业界数据显示，引入有效的SLT筛选后，可额外拦截 **500～2000 DPPM** 级别的系统级缺陷，视产品复杂度及前期测试覆盖而异。这些缺陷往往不是DC/AC参数不合格，而是时序边缘、信号完整性退化、软件操作序列或跨电源域引发的偶发性故障。对要求<10 DPPM的车规或零缺陷应用，这笔投入是必须的安全垫。

> DPPM（Defective Parts Per Million）是衡量交付质量的直接指标，每一颗到客户端的失效都可能引发换货甚至安全召回，PTE要习惯用DPPM倒推测试覆盖率是否充足。
